Complete guide to Flint configuration files and options.

## Configuration Hierarchy

Flint uses layered configuration:

1. **Global** - `~/.flint/config.toml` - User defaults
2. **Workspace** - `flint.toml` - This Flint's configuration
3. **Lock** - `.flint/flint.lock` - Computed state (don't edit)

Lower layers override higher layers.

## flint.toml

The main configuration file for your Flint.

### Basic Structure

```toml
[flint]
name = "my-project"
type = "flint"

[plugins]
required = [
  "living-documents",
  "increments",
  "projects"
]

[mods]
required = [
  "git",
  "claude-code"
]
```

### [flint] Section

Core workspace configuration.

```toml
[flint]
name = "my-project"           # Required: folder/registry name
type = "flint"                # flint | subflint
```

| Field | Required | Description |
|-------|----------|-------------|
| `name` | Yes | Unique identifier, matches folder name |
| `type` | Yes | Workspace type (`flint` or `subflint`) |

### [plugins] Section

Declare installed plugins using the `required` array.

```toml
[plugins]
required = [
  "core",
  "living-documents",
  "increments",
  "projects",
  "archive"
]
```

Plugins are specified by their kebab-case names. Use `flint plugin list --available` to see all available plugins.

**Available plugins:**
- `core` - Core conventions
- `living-documents` - Document lifecycle tracking
- `increments` - Version stream management
- `projects` - Task and backlog management
- `archive` - Archival operations
- `notepad` - Quick capture
- `comments` - Note discussions
- `flint-terminal-agent` - Agent conventions
- `autonomy` - Autonomous workflows
- `testing` - Test tracking
- `compiler` - Export compilation
- `connections` - Connection management
- `models` - Model tracking
- `documentation` - Documentation generation
- `timelines` - Timeline visualization

### [mods] Section

Declare installed mods using the `required` array.

```toml
[mods]
required = [
  "git",
  "claude-code"
]
```

**Available mods:**
- `git` - Git integration with `.gitignore` management
- `claude-code` - Claude Code integration with `CLAUDE.md`

### [workspace] Section

External resource references (machine-specific, gitignored).

```toml
[workspace]
monorepo = { path = "/Users/nathan/dev/nuu/main" }
design-docs = { path = "/Users/nathan/design/docs" }
```

This section is typically gitignored since paths are machine-specific.

### [imports] Section

Import definitions from other Flints.

```toml
[imports]
meshes = [
  { name = "Architecture", export = "api-bundle" }
]
```

## Global Configuration

User-wide settings in `~/.flint/config.toml`.

```toml
[open]
apps = ["obsidian", "cursor"]
```

### Available Options

| Option | Default | Description |
|--------|---------|-------------|
| `open.apps` | `[]` | Applications to open Flints in |

The global config is minimal. Most settings are per-workspace in `flint.toml`.

## Lock File

`.flint/flint.lock` tracks computed state. **Do not edit manually.**

```yaml
version: 1
flintVersion: "1.0.0"
created: 2025-12-18T09:00:00.000Z
synced: 2025-12-18T10:00:00.000Z

plugins:
  - name: living-documents
    version: "1.0.0"
    installedAt: 2025-12-18T09:00:00.000Z

mods:
  - name: git
    version: "1.0.0"
    installedAt: 2025-12-18T09:00:00.000Z

imports:
  - mesh: "(Mesh) Architecture"
    meshId: "uuid-here"
    export: "api-bundle"
    path: "Imports/(Import) Architecture/api-bundle"
    lastSync: 2025-12-18T10:00:00.000Z
```

The lock file:
- Tracks what's actually installed
- Records versions and timestamps
- Stores import state
- Regenerated by `flint sync`

## Connections

Connections to other Flints are stored in `.flint/connections.json`, not in `flint.toml`.

```json
{
  "version": 1,
  "connections": [
    {
      "name": "Architecture",
      "path": "/path/to/architecture-flint",
      "created": "2025-12-18T10:00:00.000Z"
    }
  ]
}
```

Use `flint connection add <name>` to add connections. The target must be a registered Flint.

## Subflints

Subflints are tracked in `.flint/subflints.json`.

```json
{
  "version": 1,
  "subflints": [
    {
      "name": "Docs",
      "relativePath": "Subflints/(Flint) Docs",
      "path": "/full/path/to/subflint",
      "created": "2025-12-18T10:00:00.000Z",
      "preset": "blank"
    }
  ]
}
```

## Plugin Configuration

Plugins are configured through their init files (`init-[id].md`) and setup skills (`setup-[id].md`). There is no separate config file - plugin behavior is defined by markdown prompts that agents interpret.

See each plugin's init file for its conventions and options.

## .gitignore

Flint manages `.gitignore` through the git mod. Default entries:

```text
# === Flint Managed (do not edit) ===
.flint/

# Machine-specific workspace references
Workspace/*/

# Keep workspace reference docs
!Workspace/*.md

# OS files
.DS_Store
Thumbs.db
# === End Flint Managed ===
```

Run `flint git sync` to regenerate after configuration changes.

## File Frontmatter

Every markdown file can have YAML frontmatter:

```yaml
---
id: uuid-v4                    # Required (auto-generated)
tags:                          # Optional
  - "#tag1"
  - "#tag2"
---
```

### Auto-managed Fields

| Field | Description |
|-------|-------------|
| `id` | UUID, added by `flint sync` or `mesh index` |

### Common User Fields

| Field | Description |
|-------|-------------|
| `tags` | Array of tags |
| `template` | Source template name |

Note: Fields like `created`, `modified`, `type`, and `status` are stored in the index, not in file frontmatter.

## Configuration Patterns

### Minimal Flint

```toml
[flint]
name = "notes"
type = "flint"
```

### Software Project

```toml
[flint]
name = "my-app"
type = "flint"

[plugins]
required = [
  "core",
  "living-documents",
  "increments",
  "projects",
  "archive",
  "flint-terminal-agent"
]

[mods]
required = [
  "git",
  "claude-code"
]

[workspace]
codebase = { path = "/path/to/code" }
```

### Research Project

```toml
[flint]
name = "research"
type = "flint"

[plugins]
required = [
  "core",
  "living-documents",
  "notepad",
  "archive"
]

[mods]
required = [
  "git"
]
```

### Documentation Subflint

```toml
[flint]
name = "docs"
type = "subflint"
```

Subflints inherit from parent and typically have minimal configuration.
